# Helixとは
[helix](https://helix-editor.com/)とはvim, neovimと同じでTUIのテキストエディタです。
neovimはlsp, treesitter, file pickerなどを自分でプラグインとしてインストールする必要がありますが、helixには最初からこれらが同梱されており、helixをインストールするだけである程度のエディタ操作はできます。

デメリットは未だプラグインシステムが公式で存在しないことです。
プラグインシステムの提案が[こちらのpull request](https://github.com/helix-editor/helix/pull/8675)でされています。
今回は手元でこのpull requestをmasterにマージしてビルドすることで、helixでプラグインを利用してみます。

# ソースコードの準備
1. プラグインシステムのpull requestを手元でマージする
```bash
git clone git@github.com:helix-editor/helix.git
cd helix
git fetch origin pull/8675/head:pr-8675
git merge pr-8675
```
2. helix-term/Cargo.tomlの設定を以下のように書き替える。
```toml
[features]
default = ["git", "steel"]
```
3. ビルドする
```bash
cargo install --profile opt --config 'build.rustflags="-C target-cpu=native"' --path helix-term --locked
```
4. ソースコードレポジトリのruntime配架にtree sitterのgrammerなどが配置されているので、そこに対してシンボリックリンクを貼る。
```bash
ln -Tsf $PWD/runtime ~/.config/helix/runtime
```

5. 目的のプラグインを探す。たとえば[ここから](https://github.com/npupko/awesome-helix)。今回はflash.hxをインストールしてみる。

6. https://github.com/shybovycha/flash.hx からflash.scmをコピーして`~/.config/helix`に配置する。
7. `~/.config/helix/init.scm`を作成し、以下のように記述する。
```
  (require "flash.hx")
```
8. これでflash.hxというプラグインを利用できるようになっているはず。helixを開いて`:flash`と入力してコマンドが表示されれば成功。
